6. DEPARTMENT-BASED FULFILLMENT
==================================================

عند اعتماد الطلب من مشرف الفرع:

1. يتم تغيير حالة الطلب إلى approved.
2. يتم تحويله إلى fulfillment queue.
3. مسؤول التوزيع يقسم الطلب حسب الأقسام.
4. يتم إنشاء Work Order مستقل لكل قسم.
5. كل Work Order يحتوي فقط الأصناف التابعة لذلك القسم.

مثال:

Order #1001:
- تورتة فراولة
- معجنات مشكلة

النظام ينشئ:

Work Order #WO-001
Production Center: المصنع
Department: قسم التورت
Items:
- تورتة فراولة

Work Order #WO-002
Production Center: المصنع
Department: قسم المعجنات
Items:
- معجنات مشكلة

الطلب الرئيسي لا يصبح Ready إلا إذا كل أوامر العمل التابعة له أصبحت Ready.

9. DELIVERY BATCH SYSTEM
==================================================

بعد اكتمال الإنتاج وتحول الطلب إلى ready:

لا يتم إسناد الطلب مباشرة للسائق.

يتم تجميع الطلبات حسب فرع الاستلام.

مثال:

Order #1 -> فرع الشرايع
Order #2 -> فرع الشرايع

يتم إنشاء:

Delivery Batch #B001
Destination Branch: فرع الشرايع
Orders:
- Order #1
- Order #2

ثم يتم إسناد Batch للسائق.

مثال آخر:

Order #1 -> فرع الشرايع
Order #2 -> فرع الشرايع
Order #3 -> فرع العوالي
Order #4 -> فرع العوالي
Order #5 -> فرع العوالي

النظام ينشئ:

Batch #B001 -> فرع الشرايع
- Order #1
- Order #2

Batch #B002 -> فرع العوالي
- Order #3
- Order #4
- Order #5

ثم:
Driver A -> Batch #B001
Driver B -> Batch #B002

12. ACCOUNTING AUTOMATION
==================================================

عند اعتماد الطلب:
- أنشئ Sales Order تلقائيًا إذا كانت البيانات مكتملة.

عند وصول الطلب إلى Ready:
- أنشئ Draft Sales Invoice تلقائيًا.

عند Delivered:
- جهّز الفاتورة للترحيل.
- إذا كانت كل البيانات مكتملة، يمكن Submit Invoice تلقائيًا أو تركها للمحاسب للمراجعة حسب الإعدادات.

عند التحصيل:
- أنشئ Draft Payment Entry.
- المحاسب يراجع ثم يعتمد.

المحاسب مسؤول عن:
- Review
- Submit
- Reconcile
- Close Financial Day

وليس مسؤولًا عن التشغيل اليومي.

26. IMPORTANT IMPLEMENTATION RULES
==================================================

- لا تستخدم hardcoded roles داخل الصفحات.
- استخدم permissions constants.
- اجعل كل Role يعتمد على Permissions.
- لا تعتمد على Flutter فقط لحماية الصلاحيات.
- تحقق من الصلاحيات في Backend.
- اجعل الطلب قابلًا للتقسيم حسب الأصناف والأقسام.
- لا تجعل الطلب Ready إلا إذا كل الأقسام انتهت.
- لا تربط السائق بالطلب مباشرة، اربطه بـ Delivery Batch.
- اجعل المحاسب مراجع ومعتمد مالي وليس مشغل تشغيل يومي.
- استخدم Audit Logs في كل العمليات المهمة.
- حافظ على التوافق مع ERPNext.
